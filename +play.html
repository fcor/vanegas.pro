+Play

+Play website

+Play is a creative studio based in Bogotá, Colombia. They wanted to share their work in a clean and unique way, So they contact us to develop together their digital portfolio.

The Challenge

  • Develop the website

  • Load many assets without affecting performance

UI process

Our first step in every project is to organize all the info and generate the IA (information arquitecture). Then with this info we define each screen for the website and this is were we start our wireframing process.

In this process we just use pen and paper to define the structure of the web page, based in the collected information. This step is were we create the structure of the site.

Digitalizing the wireframes

In this step we just clean up the wireframes to get closer to the final structure.

Foundations

+Play's team provided us their foundations, they defined the fonts and colors that we need to use in their website. Finally we can merge all this things with the assets provided by +Play to create the mock up and start making the website´s look and feel.

Assets loading

In all this process until the mockups were done, I talked a lot with the developer to understand what things we can and can’t do. Both of us identified that the video assets were too heavy for the website, so we needed to find a solution for this problem. At first, we thought using gifs directly, but we immediately found that this approach had many issues. Gif files doesn't have good quality, are not performant on some devices and using classic image html element didn't provide us full control over the media files and that would be a problem for the mobile version. We ended up using mp4 files with html 5 video and optimizing them using ffmpeg. This solution wouldn't be possible without working closely with the developer. We use the lazy loading technique to just load the image that the user is viewing.
